Finora Setup & Customization Guide

Official setup guide for the Finora Flutter Personal Finance App

1. Introduction

Finora is a production-ready, offline-first personal finance app built with Flutter. All data is kept securely on the user's device using Hive database storage.

Highlights:
  • No cloud backend required — zero server cost.
  • Robust state management via Bloc architecture.
  • Advanced local features: monthly budgets, customized categories, CSV data export, and PDF trends.
  • Biometric/PIN Security using device credentials.

2. Environment Setup

Before launching the code, ensure you have the proper Flutter development environment set up:

  1. Install Flutter SDK:

    Download and install the Flutter SDK (version 3.6.0 or higher) from the official website. Follow installation steps for your Operating System.

  2. Install your IDE:

    We recommend using VS Code (with Flutter and Dart extensions) or Android Studio.

  3. Verify Environment:

    Run the doctor tool in your command line to verify your setup is complete:

    flutter doctor

3. Running the Project

Once your environment is ready, open the project workspace in your editor and follow these commands:

  1. Get Packages:

    Install all project dependencies declared in the pubspec file:

    flutter pub get
  2. Run on Emulator/Device:

    Launch the app on a connected Android/iOS device or desktop emulator:

    flutter run

4. Rebranding & Customization

To brand this app as your own or customize it for your clients, modify the following core values:

4.1 Global Metadata Config

You can change the App Name, bundle identifier, default themes, and app configuration values in a single centralized branding file located at:

📁 lib/core/constants/app_branding.dart

class AppBranding {
  AppBranding._();

  static const displayName = 'Finora'; // Change to your custom name
  static const applicationId = 'com.finora.app'; // Update package ID
  static const version = '1.0.0';
  static const primaryColorHex = '#00B2E7'; // Your brand primary color
}

4.2 Global Package Name Rename

Important:

To publish the app on Google Play or App Store, you must change the native package name (from com.finora.app) to your own domain.

We recommend using the standard package renamer tool. Add the package to dev dependencies or run it directly:

dart run change_app_package_name:main com.yourcompany.appname

4.3 Regenerating Launcher Icons

To change the launcher icons across all platforms (Android, iOS, Web):

  1. Replace the template icon file at assets/icon/app_icon.png with your own logo (1024x1024 px).
  2. Run the following generator command in your project terminal:
dart run flutter_launcher_icons

5. Code Architecture

Finora is built using Clean Architecture practices combined with robust state management:

Layer Location Role
Core lib/core/ Themes, custom export tools (PDF/CSV), app locks, database setup.
Data lib/data/ Hive schemas, data persistence entities, mapping repos.
Features lib/features/ Visual flows (Dashboard, Budget, Transactions) containing Bloc files and UI Pages.

5.1 Regenerating Hive TypeAdapters

If you modify any database models (e.g. adding new fields to BudgetModel or TransactionModel in lib/data/models/), you must rebuild the Hive local database adapters by running:

dart run build_runner build --delete-conflicting-outputs

6. Building & Release

Follow these steps to produce the final release packages for store deployment.

6.1 Android Keystore & Build

Before launching a production APK or App Bundle, you must sign it:

  1. Generate a secure keystore file using terminal keytool.
  2. Create a file named key.properties in the android/ directory to map configuration aliases.
  3. Build the final Google Play Bundle (.aab):
flutter build appbundle --release

6.2 iOS App Store Build

Open the ios/Runner.xcworkspace in Xcode on a macOS system:

  1. Select your developer Signing & Capabilities profile under targets.
  2. Compile the release archive:
flutter build ipa --release
